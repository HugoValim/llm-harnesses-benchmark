# Issue tracker - GitHub

Issues for this repo live in [GitHub Issues](https://github.com/HugoValim/llm-harnesses-benchmark/issues). Skills that read or write issues use the [`gh`](https://cli.github.com/) CLI.

## Repository

- Owner/repo: `HugoValim/llm-harnesses-benchmark`
- Remote: `git@github.com:HugoValim/llm-harnesses-benchmark.git`
- Entry label for new agent-created issues: `needs-triage`

## Conventions

- Create: `gh issue create --title "..." --body "..."` (use `-F -` for stdin bodies).
- Read: `gh issue view <n>` for a single issue, `gh issue list` for browsing.
- Comment: `gh issue comment <n> --body "..."`.
- Label: `gh issue edit <n> --add-label "..." --remove-label "..."`.
- Close: `gh issue close <n> --reason "completed|not planned"`.
- AI-generated comments must start with:

```markdown
> *This was generated by AI during triage.*
```

## Authentication

`gh auth status` must show an authenticated user with `repo` scope. If not, prompt the maintainer to run `gh auth login` rather than attempting workarounds.

## Useful queries

- Unlabelled issues: `gh issue list --search "no:label is:open"`
- Needing triage: `gh issue list --label "needs-triage"`
- Ready for agents: `gh issue list --label "ready-for-agent"`

Replace label names with whatever is configured in `triage-labels.md` if the defaults are changed later.
